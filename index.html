What is a UTM link? (30-second explainer)

A UTM link is just a normal URL with a few extra tags on the end that tell Google Analytics where a visitor came from. Nothing is “registered” anywhere — the tags are read automatically the moment someone clicks the link and lands on the site.

https://example.plrd.org/insights/post/?utm_source=twitter&utm_medium=social&utm_campaign=post

Everything after the ? is the tagging. Without it, that visit just shows up as “Direct / unknown” in GA4. It works on any domain that has an analytics tag — plrd.org, protocol.ai, etc. GA reads the tags into whatever property that site uses.

The tags, in plain English

TagAnswers…Examples
utm_sourceWhere is it posted? (the specific place)twitter, linkedin, telegram, newsletter
utm_mediumWhat type of channel is that? (the category)social, email, referral
utm_campaignWhich push / post / series is this?human-connectome
utm_content(optional) which version / part of the link?part-1, thread-1, bio-link

Wait — what exactly is “medium”?

Source is the exact place (“twitter”). Medium is the bucket that place belongs to (“social”). Twitter, LinkedIn, Telegram and Bluesky are all different sources, but they’re the same medium = social. Your newsletter is source newsletter, medium email.

Why it matters: GA4 groups traffic into channels (Organic Social, Email, Referral…) based on the medium. If you type the wrong medium, GA4 can’t bucket it and the traffic falls into “Unassigned.” So keep it to social (any social post), email (newsletter), referral (a partner site linking us), or offline (a QR code on slides / print — see below).

QR codes on slides & print (offline)

Putting a QR on a presentation, poster, or handout? Use medium offline (source presentation). GA4 has no default “offline” channel, so these land under Unassigned in the default channel group — that’s expected. You’ll still see them clearly via the Session source / medium dimension (e.g. presentation / offline), and you can add a custom channel group later if you want a dedicated bucket.

Publishing a blog series?

Use utm_campaign for the series (e.g. connectome-series) so every part is grouped together in GA4. Use utm_content for the part — part-1, part-2, part-3 — so the individual posts are distinguishable right in the campaign report.

Note: each post already has its own landing-page URL, so GA can tell them apart anyway — but putting the part in utm_content makes it readable without cross-referencing URLs. Just give utm_content one job per link (the part or a placement like bio-link, not both).

How to use it

1 · Paste the page link.   2 · Pick the channels you’ll post to.   3 · Copy each tagged link (or grab its QR) into that post/newsletter.   4 · A few days later, see results in GA4 → Reports → Acquisition → Traffic acquisition (set the dimension to Session campaign to compare channels for one post/series).

Golden rule: reuse the same campaign name across all channels for one push, and only tag external links pointing to the site — never internal page-to-page links.
